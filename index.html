扩展VLAN特性配置与管理

VLAN聚合：

VLAN聚合是把多个不配置三层VLANIF接口聚合成一个超级VLAN（Super-VLAN），这些VLANIF的接口称为Sub-VLAN

Super-VLAN：
可以看成一个大的VLAN，但它的成员不是端口而是Sub-VLAN，需要配置三层VLANIF接口并配置IP地址。每个VLAN聚合中只能有一个Super-VLAN。

Sub-VLAN：
它是Super-VLAN的成员，它的成员是交换机的端口，不能创建三层VLANIF接口，不同Sub-VLAN间是相互二层隔离的，但各个Sub-VLAN中的用户网关都是Super-VLAN的VLANIF接口IP地址，来实现各Sub-VLAN成员间以及与外部网络的三层通信。每个VLAN聚合中可以有多个Sub-VLAN。

MUX VLAN：

MUX VLAN（Multiplex VLAN）提供了一种通过VLAN进行网络资源控制的机制

  1. 主VLAN（Principal VLAN）：Principal port可以和MUX VLAN内的所有接口进行通信。
  2. 从VLAN（Subordinate VLAN）包括隔离型从VLAN和互通型从VLAN：
  3. 隔离型从VLAN（Separate VLAN）：Separate port只能和Principal port进行通信，和其他类型的接口实现完全隔离。

每个隔离型从VLAN必须绑定一个主VLAN。
互通型从VLAN（Group VLAN）：Group port可以和Principal port进行通信，在同一组内的接口也可互相通信，但不能和其他组接口或Separate port通信。
每个互通型从VLAN必须绑定一个主VLAN。

QINQ技术：

QinQ封装过程就是把单层802.1Q标签的数据帧转换成双层802.1Q标签的数据帧，可以分为基本QinQ和灵活QinQ两种封装类型

基本QinQ封装是把进入一个端口的所有流量都封装一个相同的外层VLAN标签，也称为“QinQ二层隧道“
灵活QinQ封装是基于端口和VLAN封装的结合方式，对同一个端口接收的数据帧可以根据不同的内层VLAN标签封装不同的外层标签。灵活QinQ又可以分为以下3种类型：

  1. 基于VLAN ID的灵活QinQ：基于不同的内层VLAN ID的标签来封装不同的外层标签
  2. 基于802.1p优先级的灵活QinQ：基于不同的内层802.1p的标签来封装不同的外层标签
  3. 基于流策略的灵活QinQ：根据不同的QoS策略来添加不同的外层标签

QinQ/Dot1q终结子接口：是指设备对数据帧的双层或单层VLAN标签进行识别，根据后续的转发行为进行剥离外层标签，然后据需传送。

QinQ映射：QinQ映射是在路由子接口上应用的，且发生在数据帧接收进来之前和从端口发送出去前。向外发送本地VLAN帧时，将帧中的本地VLAN标签替换成外部VLAN标签；接收外部VLAN的帧时，又将外部VLAN标签替换成本地VLAN标签。

VLAN映射：

VLAN映射不同于QinQ映射应用方式，VLAN映射是在物理端口上应用的

配置1 to 1的VLAN映射

  1. 基于VLAN的1 to 1的VLAN映射：可以在接收到带有单层VLAN标签的数据帧后，将帧中的VLAN ID映射为指定的VLAN ID。
  2. 基于802.1p优先级的1 to 1的VLAN映射：可以根据接收到的数据帧的802.1p优先级和VLAN ID进行映射，且是在入端口配置的。
  3. 基于流策略的1 to 1的VLAN映射：流策略是指将流分类后形成的完整的QoS策略，分为以下4个步骤：a)根据数据帧中的VLAN ID进行流分类； b)定义基于VLAN ID的流行为； c)创建QoS策略，将流分类与流行为进行关联； d)在端口上应用QoS策略。

接口及以太网链路配置及管理

以太网接口属性：

二层接口是一种物理接口，属于数据链路层，不能配置IP地址，可以对接收到的报文经行二层转发。

三层子接口是一种逻辑接口，属于网络层，可以配置IP地址，主要用来实现收发VLAN报文。可以通过配置使一个二层接口绑定多个三层子接口，从而使得不同VLAN报文从不同子接口转发。

接口类型：

接口类型分为管理接口、物理接口和逻辑接口
管理接口用于登陆到设备经行配置与管理，此接口不承担业务传输
物理接口是真实存在的可承担业务传输的接口，一般由电口与光口组成
逻辑接口是指能实现数据交换但物理上不存在的接口

端口隔离

端口隔离分为单向隔离和端口隔离组，且都仅适用于同意交换机上不同端口间的隔离，且一个端口可以加入多个端口隔离组

单向隔离指禁止某个端口发送的报文到达其他端口，而不禁止其他端口的报文到达本端口
而端口隔离组内的接口则是互相隔离的

链路聚合

链路聚合是指将多个相同属性的物理接口捆绑在一起形成一个逻辑接口，可以用来增加带宽，且聚合端口与物理接口同样可以配置端口类型。

负载分担：

负载分担分为手动负载分担和静态LACP负载分担。在手动负载分担的模式下，聚合端口与成员接口的配置都需要由手工来经行完成，并且在该模式下所有链路都参与数据的转发，平均分担流量；静态LACP负载分担模式则是利用LACP协议进行自动聚合，通过LACPDU报文经行交互，通告自己的（系统优先级、系统MAC、端口优先级、端口号和操作key）并与对端经行比较，选择能够汇聚的端口，链路的两端称为Actor和Partner，且LACP模式可以实现链路冗余备份（所以也被称为M:N模式）。

LACP建立主被动关系顺序：
1、系统优先级（system priority）越低越优先，缺省值都为32768
2、系统MAC（system ID），越小越优先

注意事项：
成员接口的链路类型（华为默认为Hybrid）、所属VLAN、接口优先级、接口类型（百兆千兆）等需为一致
成员接口不能配置任何业务及静态MAC地址
一个以太网接口只能属于一个聚合端口中，如果需要加入其他聚合端口需先推出原来的聚合口
两端的互联接口都应配置为聚合端口，否则不能正常通信
聚合端口两端的成员接口数、速率、双工模式等属性配置需一致

负载分档方式：
dst-ip（目的IP）：同一个源IP发到不同目的IP的数据包将在不同端口发送，不同源地址但同目的IP地址的数据包总在同个端口发送
dst-mac（目的mac）：同一个源MAC发到不同目的MAC的数据包将在不同端口发送，不同源MAC但同目的MAC的数据包总在同个端口发送
src-ip（源IP）：来自不同IP地址的数据包总在不同端口转发，同一源IP的数据包总在同个端口转发
src-mac（源MAC）：来自不同MAC的数据包总在不同端口转发，同一源MAC的数据包总在同个端口转发
src-dst-ip（源与目的IP）：在不清楚是基于源IP转发还是目的IP转发时适用，AtoB BtoC CtoB都在不同端口转发
src-dst-mac（源与目的MAC）：在不清楚是基于源MAC转发还是目的MAC转发时适用，AtoB BtoC CtoB都在不同端口转发

LACP抢占是指当链路中的高优先级端口出现故障时低优先级端口能代替故障端口经行转发
且LACP抢占有抢占延迟，指切换端口需要等待的时间
LACP报文超时时间是指当某个成员端口出现故障，则在超时时间后未收到LACP报文则认为对端不可达，不再转发数据

基本VLAN特性配置与管理

VLAN基础

VLAN（虚拟局域网）是一个将物理LAN逻辑划分为若干个虚拟的LAN，目的是为了缩小广播域，使广播域进行二层（数据链路层）通信，不同VLAN间的主机是不能互相通信的，必须经过第三层（网络层）进行通信。
VLAN既能限制广播网络，又可增强局域网的安全性，还提高了网络的健壮性。
在同一个物理交换机上不可能存在两个相同的VLAN，在不同交换机上则可以有相同VLAN

VLAN帧

VLAN Tag:标识所属VLAN，遵循IEEE 802.1Q标注，支持802.1Q协议的交换机既可以发送有标记帧也可以发送无标记帧（有标记帧加入了4字节的801.1Q Tag，反之则没有），交换机内部的数据包一律携带Tag。

802.1Q Tag各字段含义：

TPID：长度2bytes，表示帧类型，取值为0x8100为802.1Q帧，如果不支持的设备接受到则丢弃
PRI：长度3bits，表示帧优先级，取值范围0-7，越大越优，用于QOS
CFI：长度1bits，表示MAC地址是否为经典格式，0是经典1是非经典，在以太网中值为1
VID：长度12bits，表示VLAN ID

VLAN特性

华为交换机支持5种VLAN划分：基于端口划分、基于子网划分、基于MAC地址划分、基于协议划分、基于策略划分；其中端口划分属于静态VLAN划分，其余的属于动态VLAN划分
（除了端口划分其余都要在Hybrid口上划分）

常见的VLAN间通信有三层VLANIF接口、三层以太网子接口和VLAN switch方案

VLAN聚合：是指Super VLAN技术，他是在一个主VLAN下包括多个同IP网段的VLAN技术，但只需要为主VLANIF配置IP地址，各从VLAN中的用户主机就可以通过主VLAN的VLANIF接口作为缺省网关进行通信

VLAN映射：在一些交换机上不支持全范围（1-4094）的VLAN ID，而只支持1-512的VLAN ID就与公网的VLAN ID实现冲突；于是就产生了一种可以实现在用户VLAN ID与运营商VLAN ID进行相互转换的技术。

语音VLAN：（Voice VLAN）是相对传统VLAN数据而言的，他是针对不同类型的用户语言流进行划分的，并能自动修改数据帧的优先级，以提高语音数据的传输质量

VLAN透传：可以使交换机直接透明传输指定VLAN的数据帧，不上送CPU处理，也不用去识别VLAN标签，提高了转发效率

QINQ协议：是基于IEEE 802.1Q技术的一种二层隧道协议。在公网中传输的数据一般分为公网VLAN标签和私网VLAN标签，QINQ的核心思想是为了将私网标签封装在公网标签中，为用户提供一种较为简单的二层VPN隧道

二层以太网端口

二层以太网端口类型有：access、trunk、hybrid和QINQ四种端口类型

Access端口：主要用来连接用户主机的二层以太网端口，仅允许一个VLAN通过，发送的以太网帧永远是不带标签的（Untagged）
Trunk端口：主要用来连接与其他交换机的二层端口，允许多个VLAN通过，发送的帧都是带标签的
Hybrid端口：Hybrid端口可以说是access端口和trunk端口的混合体，是一种特殊的二层以太网端口，既可以连接主机也可以连接交换机，可以允许一个或多个VLAN通过，并可以选择带标签或不带标签。
QINQ端口：QINQ端口是专用于QINQ协议的二层以太网端口，可以给数据帧加上双层VLAN标签，从而可以支持多达4094*4094个VLAN。

二层以太网链路

以太网链路分为接入链路（Access link）和干道链路（Trunk link）

接入链路：是交换机直接连接用户主机的链路。通常情况下主机并不需要识别自己是属于哪个VLAN，所以接入链路发送和接受的数据帧都是不带标签（Untagged）的，但接入链路不一定只允许一个VLAN通过，只是access端口才仅允许一个VLAN通过，当属于Hybrid端口时同样是允许多个VLAN通过的

干道链路：是交换机与交换机互联的链路。可以承载多个不同VLAN数据传输，干道链路两端的设备需要识别VLAN标签，所以在干道链路的数据都是带标签（Tagged）的，除了该链路的PVID所属VLAN（默认VLAN1）

VLAN间通信

  1. 三层VLANIF接口方案（VLANIF接口作为对应VLAN内的用户缺省网关）
  2. 三层以太网子接口方案（同时具备三层以太网物理接口和二层以太网物理接口的双重特性）
  3. VLAN Switch方案（一种按照VLAN标签进行数据转发的技术，需要预先在各交换机上配置静态转发路径，符合条件即可转发到对应的出接口，无需查看MAC地址，提高转发效率和安全性，能避免MAC地址攻击和广播风暴）

# 链表
class Node:
    def __init__(self, item=None):
        self.item = item
        self.next = None
        self.prior = Node  # 双链表

# 头插法
def create_linklist_head(lst):
    head = Node(lst[0])
    for element in lst[1:]:
        node = Node(element)
        head.prior = node  # 双链表
        node.next = head
        head = node
    return head

# 尾插法
def create_linklist_tail(lst):
    head = Node(lst[0])
    tail = head
    for element in lst[1:]:
        node = Node(element)
        tail.next = node
        tmp = tail
        tail = node
        tail.prior = tmp  # 26&28双链表
    return head

# 输出链表
def print_linklist(lk):
    while lk:
        print(lk.item, end=',')
        lk = lk.next
    print()

# 头插法eg
lk1 = create_linklist_head([1, 2, 3])
print(lk1.item)  # 3
print(lk1.next.item)  # 2
print(lk1.next.prior.item)  # 3(双链表)
print_linklist(lk1)  # 3,2,1,
# 尾插法eg
lk2 = create_linklist_tail([1, 2, 3])
print(lk2.item)  # 1
print(lk2.next.item)  # 2
print(lk2.next.prior.item)  # 1(双链表)
print_linklist(lk2)  # 1,2,3,
# 插入元素
p = Node(5)
p.next = lk2.next
lk2.next.prior = p
p.prior = lk2   # 54&55双链表
lk2.next = p
print_linklist(lk2)  # 1,5,2,3,
# 删除元素
lk2.next.next.prior = lk2
lk2.next = lk2.next.next
print_linklist(lk2)  # 1,2,3,

插入&删除：   O(n)      O(1)

# AVLTree
# https://www.bilibili.com/video/BV1uA411N7c5?p=74&vd_source=2493c37afb3b31338e613a5ec70a3fcd
from BST import BiTreeNode, BST

class AVLNode(BiTreeNode):
    def __init__(self, data):
        BiTreeNode.__init__(self, data)
        self.bf = 0

class AVLTree(BST):
    def __init__(self, li=None):
        BST.__init__(self, li)

    def rotate_left(self, p, c):
        lc = c.lchild
        p.rchild = lc
        if lc:
            lc.parent = p
        c.lchild = p
        p.parent = c
        p.bf = 0
        c.bf = 0
        return c

    def rotate_right(self, p, c):
        rc = c.rchild
        p.lchild = rc
        if rc:
            rc.parent = p
        c.rchild = p
        p.parent = c
        p.bf = 0
        c.bf = 0
        return c

    def rotate_right_left(self, p, c):
        g = c.lchild
        rrc = g.rchild
        c.lchild = rrc
        if rrc:
            rrc.parent = c
        g.rchild = c
        c.parent = g
        rlc = g.lchild
        p.rchild = rlc
        if rlc:
            rlc.parent = p
        g.lchild = p
        p.parent = g
        if g.bf > 0:
            p.bf = -1
            c.bf = 0
        elif g.bf < 0:
            p.bf = 0
            c.bf = 1
        else:
            p.bf = 0
            c.bf = 0
        return g

    def rotate_left_right(self, p, c):
        g = c.rchild
        rlc = g.lchild
        c.rchild = rlc
        if rlc:
            rlc.parent = c
        g.lchild = c
        c.parent = g
        rrc = g.rchild
        p.lchild = rrc
        if rrc:
            rrc.parent = p
        g.rchild = p
        p.parent = g
        if g.bf > 0:
            p.bf = 0
            c.bf = -1
        elif g.bf < 0:
            p.bf = 1
            c.bf = 0
        else:
            p.bf = 0
            c.bf = 0
        return g

    def insert(self, val):
        p = self.root
        if not p:
            self.root = AVLNode(val)
            return
        while True:
            if val < p.data:
                if p.lchild:
                    p = p.lchild
                else:
                    p.lchild = AVLNode(val)
                    p.lchild.parent = p
                    node = p.lchild
                    break
            elif val > p.data:
                if p.rchild:
                    p = p.rchild
                else:
                    p.rchild = AVLNode(val)
                    p.rchild.parent = p
                    node = p.rchild
                    break
            else:
                return
        while node.parent:
            g = node.parent.parent
            x = node.parent
            if node == node.parent.lchild:
                if node.parent.bf < 0:
                    if node.bf > 0:
                        n = self.rotate_left_right(node.parent, node)
                    else:
                        n = self.rotate_right(node.parent, node)
                elif node.parent.bf == 0:
                    node.parent.bf = -1
                    node = node.parent
                    continue
                else:
                    node.parent.bf = 0
                    break
            else:
                if node.parent.bf > 0:
                    if node.bf < 0:
                        n = self.rotate_right_left(node.parent, node)
                    else:
                        n = self.rotate_left(node.parent, node)
                elif node.parent.bf == 0:
                    node.parent.bf = 1
                    node = node.parent
                    continue
                else:
                    node.parent.bf = 0
                    break
            n.parent = g
            if g:
                if x == g.lchild:
                    g.lchild = n
                else:
                    g.rchild = n
                break
            else:
                self.root = n
                break

tree = AVLTree([9,8,7,6,5,4,3,2,1])
tree.order(tree.root)
print()
tree.insert(10)
tree.order(tree.root)
