扩展VLAN特性配置与管理

VLAN聚合：

VLAN聚合是把多个不配置三层VLANIF接口聚合成一个超级VLAN（Super-VLAN），这些VLANIF的接口称为Sub-VLAN

Super-VLAN：
可以看成一个大的VLAN，但它的成员不是端口而是Sub-VLAN，需要配置三层VLANIF接口并配置IP地址。每个VLAN聚合中只能有一个Super-VLAN。

Sub-VLAN：
它是Super-VLAN的成员，它的成员是交换机的端口，不能创建三层VLANIF接口，不同Sub-VLAN间是相互二层隔离的，但各个Sub-VLAN中的用户网关都是Super-VLAN的VLANIF接口IP地址，来实现各Sub-VLAN成员间以及与外部网络的三层通信。每个VLAN聚合中可以有多个Sub-VLAN。

MUX VLAN：

MUX VLAN（Multiplex VLAN）提供了一种通过VLAN进行网络资源控制的机制

  1. 主VLAN（Principal VLAN）：Principal port可以和MUX VLAN内的所有接口进行通信。
  2. 从VLAN（Subordinate VLAN）包括隔离型从VLAN和互通型从VLAN：
  3. 隔离型从VLAN（Separate VLAN）：Separate port只能和Principal port进行通信，和其他类型的接口实现完全隔离。

接口及以太网链路配置及管理

以太网接口属性：

二层接口是一种物理接口，属于数据链路层，不能配置IP地址，可以对接收到的报文经行二层转发。

三层子接口是一种逻辑接口，属于网络层，可以配置IP地址，主要用来实现收发VLAN报文。可以通过配置使一个二层接口绑定多个三层子接口，从而使得不同VLAN报文从不同子接口转发。

接口类型：

接口类型分为管理接口、物理接口和逻辑接口
管理接口用于登陆到设备经行配置与管理，此接口不承担业务传输
物理接口是真实存在的可承担业务传输的接口，一般由电口与光口组成
逻辑接口是指能实现数据交换但物理上不存在的接口

端口隔离

端口隔离分为单向隔离和端口隔离组，且都仅适用于同意交换机上不同端口间的隔离，且一个端口可以加入多个端口隔离组

  1. 单向隔离指禁止某个端口发送的报文到达其他端口，而不禁止其他端口的报文到达本端口
  2. 而端口隔离组内的接口则是互相隔离的

链路聚合

链路聚合是指将多个相同属性的物理接口捆绑在一起形成一个逻辑接口，可以用来增加带宽，且聚合端口与物理接口同样可以配置端口类型。

负载分担：

负载分担分为手动负载分担和静态LACP负载分担。在手动负载分担的模式下，聚合端口与成员接口的配置都需要由手工来经行完成，并且在该模式下所有链路都参与数据的转发，平均分担流量；静态LACP负载分担模式则是利用LACP协议进行自动聚合，通过LACPDU报文经行交互，通告自己的（系统优先级、系统MAC、端口优先级、端口号和操作key）并与对端经行比较，选择能够汇聚的端口，链路的两端称为Actor和Partner，且LACP模式可以实现链路冗余备份（所以也被称为M:N模式）。

基本VLAN特性配置与管理

VLAN基础

VLAN（虚拟局域网）是一个将物理LAN逻辑划分为若干个虚拟的LAN，目的是为了缩小广播域，使广播域进行二层（数据链路层）通信，不同VLAN间的主机是不能互相通信的，必须经过第三层（网络层）进行通信。
VLAN既能限制广播网络，又可增强局域网的安全性，还提高了网络的健壮性。
在同一个物理交换机上不可能存在两个相同的VLAN，在不同交换机上则可以有相同VLAN

VLAN帧

VLAN Tag:标识所属VLAN，遵循IEEE 802.1Q标注，支持802.1Q协议的交换机既可以发送有标记帧也可以发送无标记帧（有标记帧加入了4字节的801.1Q Tag，反之则没有），交换机内部的数据包一律携带Tag。

class Node:
    def __init__(self, item=None):
        self.item = item
        self.next = None
        self.prior = Node  # 双链表

# 头插法
def create_linklist_head(lst):
    head = Node(lst[0])
    for element in lst[1:]:
        node = Node(element)
        head.prior = node  # 双链表
        node.next = head
        head = node
    return head
